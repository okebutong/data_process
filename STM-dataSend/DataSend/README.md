# 大致流程

 温湿度传感器接收温湿度信息，添加帧头、数据信息和校验码，然后将合成数据放入到buf中(读写锁保护)，待可以读取的状态下时，数据通过串口发送至上位机。

获取信号量，开始读取数据
同时此处需要分为n个线程，用来同时读取

当读取完毕之后，需要将获取的内容发送至TLV数据生成模块，当模块生成数据完毕之后，将数据将其发送
n个设备，n个线程
每个线程读取完的数据，直接送往TLV数据生成模块，此处TLV生成模块需要加装读写锁，用来保护buffer,

# 嵌入式数据采集

## 项目概述

### 1.1 项目名称

嵌入式设备数据采集系统

### 1.2 项目背景

为了实现实时检测环境温湿度，开发一款基于 Stm32芯片环境动态监测系统。

系统通过串口与上位机通信，发送设备采的 TLV 格式数据。

### 1.3 开发目标

实现以下目标：

* 稳定接收温湿度数据
* 正确生成 TLV 协议
* 能够稳定发送数据

# 3. 功能需求

## 3.1 数据发送模块

### 功能描述

利用串口发送设备接收的数据。

### 输入

TLV数据包

### 输出

串口字节流

## 3.2 TLV生成模块

### 功能描述

解析设备发送的数据包。

### 输入

id号，读取的数据

### 输出

TV格式的数据

### 协议格式

```text
+--------+--------+------+---------+----------+
| header | Dev id | Type | Value   | CheckNum |
+--------+--------+------+---------+----------+
|   2    |   1    |   1  |    4    |    2     |
+--------+--------+------+---------+----------+
```

### 示例

```
输入：id 1825616515 value 24.9 
输出：AA 55 01 01 00 0A 01 F9 CA 91
```

```text
AA 55 01 01 00 0A 01 F9 CA 91
```

解析结果：

```text
    AA 55     帧头
    01        设备ID，十进制1
    01        数据类型 temp温度值
    00 0A     数据倍数，发送小数的十六进制不方便，故将数值提升至n倍
    01        Value长度，占2字节
    F9        数据值，十进制220
    CA 91     校验位
```
